## TEACHER TESTING OBJECTIVES

### Primary Goals

**1. Intervention Recommendation Efficacy** ⭐ CRITICAL
- Do AI recommendations match teacher judgment?
- Are alternatives helpful when provided?
- Do teachers prefer primary or alternatives?
- What patterns emerge in teacher choices?

**2. Evidence Capture Workflow** ⭐ CRITICAL
- Is logging observations intuitive?
- Do teachers understand what to observe?
- Is the Quick Log feature useful?
- How often do teachers manually vs. AI-capture?

**3. Platform Usability** ⭐ HIGH
- Can teachers find features intuitively?
- Is navigation clear throughout?
- Do teachers successfully plan interventions?
- What features are not being discovered?

**4. Game Engagement** ⭐ HIGH
- Do students stay on-task in games?
- Are celebration animations motivating?
- Do students understand game objectives?
- Which games are most engaging?

**5. Planning & Execution** ⭐ MEDIUM
- Does the intervention plan translate to action?
- Can teachers easily start a session?
- Are resources (curriculum, games) accessible?
- What barriers prevent plan execution?

## TEACHER FEEDBACK FORMS

### Form 1: Daily Usage Survey (5 min, daily)
**Purpose**: Track daily feature usage and satisfaction

**Questions**:
1. Which features did you use today? (multi-select)
   - [ ] Morning Brief
   - [ ] Caseload review
   - [ ] Focus Card
   - [ ] Quick Log
   - [ ] Start game session
   - [ ] View reports
   - [ ] Create intervention plan
   - [ ] AI Coach suggestions

2. Ease of use (1-5 scale):
   - Morning Brief: 1 2 3 4 5
   - Evidence logging: 1 2 3 4 5
   - Intervention planning: 1 2 3 4 5
   - Game selection: 1 2 3 4 5

3. What was the biggest friction point today?
   - Open response (text)

4. Did AI recommendations help you decide? Yes / No / Partially
   - If no: What would have helped?

### Form 2: Weekly Reflection (15 min, Friday end-of-week)
**Purpose**: Deeper insights into student outcomes and feature value

**Questions**:
1. Which students showed the most progress this week?
   - And which intervention did you use with them?

2. Did the intervention recommendations match your judgment?
   - (1-5 scale: strongly disagree to strongly agree)

3. If you modified a recommendation, why?
   - [ ] Student preference
   - [ ] Schedule constraints
   - [ ] Available resources
   - [ ] Classroom setup
   - [ ] Other: ___

4. Which game did students engage with most?
   - Word Quest / Typing Quest / Precision Play / Other

5. What's one feature you'd like us to improve?
   - Open response

6. What's one feature you'd like us to add?
   - Open response

### Form 3: Two-Week Debrief (30 min, in-person or Zoom)
**Purpose**: Comprehensive assessment and recommendations

**Interview Topics**:
1. Overall impression of platform (qualitative)
2. Impact on your classroom/workflow (quantitative estimate)
3. Student outcomes - are they better/same/worse than before?
4. Top 3 features you use regularly
5. Top 3 features you never use (and why?)
6. Major pain points encountered
7. Recommendations for improvement
8. Likelihood to recommend to other teachers (1-10 scale)
9. What would make you recommend to others?
10. Additional feedback/observations

## TESTING COHORT REQUIREMENTS

### Ideal Participants
- **Count**: 5-10 teachers or specialists
- **Experience**: Mix of experienced + new-to-role teachers
- **Grade Levels**: K-5 primary, some 6-8
- **School Types**: Mix of public, charter, high-need
- **Tech Comfort**: Mix of tech-savvy + less tech-experienced
- **Commitment**: 2-3 weeks of regular use + debrief

### Recruitment Strategy
- Target schools/districts you have relationships with
- Offer light incentive (gift card? professional development credit?)
- Emphasize "shaping the product" narrative
- Highlight low barrier to entry (cloud-based, no install)
- Start with 5 teachers first 2 weeks, add 5 more based on feedback

### Logistics
- Platform deployed at: `https://bkseatown.github.io/CornerstoneMTSS/`
- Demo data / sample students provided
- Weekly check-ins via email
- Two formal surveys (daily + weekly)
- Final debrief call (30 min per teacher)
- Slack/email support for issues

## WHAT TO MONITOR DURING TESTING

### Red Flags (Stop & Investigate)
- No teachers logging in after 2 days
- >30% of teachers report major bugs
- Recommendation feature consistently rated <3/5
- Game crashes or data loss
- Mobile version completely broken
- Teachers unable to complete basic workflows

### Yellow Flags (Plan Fixes)
- Some teachers not using key features
- Mobile experience is suboptimal
- Recommendations don't always make sense to teachers
- Game engagement is lower than expected
- Teachers requesting same feature repeatedly

### Green Signals (Proceeding Well)
- Daily active user rate >70%
- Feature adoption of core features >80%
- Most teachers completing daily surveys
- Positive comments on usefulness
- High recommendation likelihood (>7/10)
- Teachers using alternatives feature
- Students engaged in games
